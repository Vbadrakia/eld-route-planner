# Deployment Guide: ELD Route Planner

This guide provides step-by-step instructions to deploy the ELD Route Planner application. 

The architecture consists of:
*   **Backend**: Django REST Framework API deployed on **Render** (via Docker & PostgreSQL).
*   **Frontend**: React (Vite) Single Page Application deployed on **Vercel**.

---

## 1. Preparing the Git Repository

Render and Vercel both deploy automatically by linking to a Git repository (like GitHub or GitLab). Because this project is located in a subfolder, you should initialize `eld_route_starter` as its own clean Git repository.

Open your terminal, navigate to the project directory, and run:

```bash
# 1. Navigate to the project directory
cd "C:\Users\VEDANT(HOME)\Desktop\Spotter\eld_route_starter"

# 2. Initialize a new Git repository
git init

# 3. Create a .gitignore file in the root if you don't have one
echo "node_modules/
venv/
db.sqlite3
.env
dist/
staticfiles/
*.pyc
__pycache__/
.DS_Store" > .gitignore

# 4. Add files and make initial commit
git add .
git commit -m "Initial commit for ELD Route Planner deployment"

# 5. Create a new repository on GitHub (e.g. named "eld-route-planner")
# then link and push to it:
git branch -M main
git remote add origin https://github.com/Vbadrakia/eld-route-planner.git
git push -u origin main
```

## 2. Backend Deployment on Render

We have preconfigured a **Render Blueprint** (`render.yaml`) which automatically spins up both the **Django Web Service** and a **PostgreSQL Database**, connecting them together.

### Steps to Deploy:

1.  Log into your [Render Dashboard](https://dashboard.render.com/).
2.  Click **New +** in the top right and select **Blueprint**.
3.  Connect your GitHub/GitLab account and select the `eld-route-planner` repository you created.
4.  Render will read the `render.yaml` file automatically:
    *   **Blueprint Name**: Choose any name (e.g., `eld-route-planner-group`).
    *   **Database**: Render will provision a free PostgreSQL instance.
    *   **Web Service**: Render will build the Docker container using `./backend/Dockerfile`.
5.  **Environment Variables**: Under the environment variables section on Render, fill in the following:
    *   `ORS_API_KEY`: *(Optional)* If you have an OpenRouteService API key, enter it here. If left blank, the app will run in Demo/Simulation mode using precomputed routing.
    *   `DJANGO_SECRET_KEY`: Automatically generated by the blueprint.
    *   `DJANGO_DEBUG`: Pre-set to `False` for production.
    *   `DJANGO_ALLOWED_HOSTS`: Pre-set to `.onrender.com`.
6.  Click **Deploy**.
7.  Once the build succeeds, copy your live API URL (e.g., `https://eld-route-planner-api.onrender.com`).

> [!NOTE]
> The backend Dockerfile is configured to automatically run database migrations (`python manage.py migrate`) and static file collection (`python manage.py collectstatic`) upon starting, so there are no manual database setup steps required!

## 3. Frontend Deployment on Vercel

The React app compiles into a static single-page application and is deployable on Vercel.

### Steps to Deploy:

1.  Log into your [Vercel Dashboard](https://vercel.com/).
2.  Click **Add New...** and select **Project**.
3.  Select your `eld-route-planner` repository.
4.  Configure the project settings:
    *   **Project Name**: `eld-route-planner`
    *   **Framework Preset**: Select **Vite** (it should auto-detect this).
    *   **Root Directory**: Set this to **`frontend`** (Click "Edit" and choose the `frontend` folder).
5.  **Environment Variables**: Expand the environment variables section and add:
    *   **Key**: `VITE_API_URL`
    *   **Value**: `https://eld-route-planner-api.onrender.com` (replace with your actual Render API service URL).
6.  Click **Deploy**.
7.  Vercel will compile the React build using `npm run build` and launch the website.

---

## 4. Post-Deployment Verification

1.  Open the Vercel URL in your browser.
2.  Type a trip (e.g., Dallas, TX to Houston, TX to Atlanta, GA, and 52 hours cycle used).
3.  Verify the Leaflet map loads, lists driving coordinates, marks breaks, displays fuel stops, and successfully renders the print-ready SVG HOS log sheets.
4.  If there are any connection issues, inspect the browser's console network requests and verify `VITE_API_URL` matches your active Render service URL.
